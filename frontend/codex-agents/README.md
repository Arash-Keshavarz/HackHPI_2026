# Codex Agents

This folder contains a minimal Codex agent setup for the agriculture hackathon dashboard.

Use `init.yaml` as the shared project snapshot for Codex. The agent files are intentionally broad.

Recommended reading order:

1. `init.yaml`
2. `project-context.md`
3. the role file for the task

Agent files:

- `ui-agent.md`
- `explorer-agent.md`
- `builder-agent.md`

Recommended usage:

- `ui-agent.md`: for dashboard layout, overlays, controls, and visual clarity
- `explorer-agent.md`: for understanding the repo, backend contract, data shape, and architecture decisions
- `builder-agent.md`: for implementing features, wiring API data, and shipping working code

Current project theme:

- Agriculture vehicle safety
- Video or frame-based dashboard
- ML outputs rendered as overlays, vectors, and explanations
- FastAPI backend
- React/Vite/TypeScript frontend
